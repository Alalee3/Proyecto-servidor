2. ARQUITECTURA GENERAL Y ESTÁNDARES ("Gold Standard": PNF y Bibliografía)
--------------------------------------------------------------------------

A. ARQUITECTURA DE DATOS (REPOSITORIOS):
- Ubicación: 'app/Repositories/[Modulo]/'
- Estructura Desacoplada: Dividir en archivos especializados (Index, Create, Edit, View).
- Implementación: Uso de 'DB::table' (Query Builder) para mayor rendimiento.
- Consistencia: Mapear datos a nombres genéricos (ej: 'nombre_pnf' as 'nombre') en el RepoIndex para uniformidad en las vistas.

B. LÓGICA DE COMPONENTES (LIVEWIRE):
- Formularios: Uso obligatorio de 'Livewire Forms' en 'app/Livewire/Forms/[Modulo]/'.
- Métodos Estandarizados:
  - Creación: 'guardar()' -> Resetea campos y muestra éxito sin redirigir (UX fluida).
  - Edición: 'actualizar()' -> Acción de guardado. 'cancelar()' -> Redirige al listado.
  - Detalle: 'cerrar()' -> Redirige al listado.
- Feedback: Validación en tiempo real con 'updated()' y 'validateOnly()'.

C. DISEÑO VISUAL (BLADE / UI):
- Listado (List):
  - No incluir botón de "Nuevo" si el acceso ya está en la barra lateral (Sidebar).
  - Uso de '<x-table.search-input />' y modales de confirmación '<x-table.delete-modal />'.
- Registro (Create):
  - Cabecera: Centrada, mayúsculas, color gris ('text-gray-500').
  - Contenedor: Blanco con sombra y bordes redondeados.
- Edición (Update):
  - Botón Volver: Usar '<x-danger-button>' con icono 'arrow_back' de Material Symbols.
- Visualización (Show):
  - Cabecera Dinámica: Título en rojo si el registro está inactivo.
  - Datos: Usar 'text-2xl font-semibold' para resaltar los valores.
  - Botón Volver: '<x-danger-button>' con icono 'arrow_back'.

3. ESTRUCTURA DE BASE DE DATOS (sogac_v_2 (2).sql)
--------------------------------------------------

A. RELACIONES Y FLUJO:
- Jerarquía: PNF -> Malla Académica -> Unidad Curricular -> Contenido -> Temas.
- Integridad: Foreign Keys obligatorias para todas las relaciones.

B. ESTÁNDARES DE TABLAS:
- Estatus: 'enum('1','2','3')' universal (1:Activo, 2:Inactivo, 3:Eliminado).
- Timestamps: 'fecha_creacion' y 'fecha_actualizacion' en todas las tablas.

4. PATRONES DE SEGURIDAD Y VALIDACIÓN
-------------------------------------
- IDs Protegidos: Atributo '#[Locked]' para evitar manipulación de claves primarias.
- Regex Universal: '/^[A-Za-záéíóúÁÉÍÓÚñÑüÜ\d\s\.,\-\(\)\"\':\/]+$/u'
  - Permite letras, números, espacios y caracteres especiales de puntuación (indispensable para Bibliografía).
- Mensajes: No usar genéricos. Definir 'messages()' en cada Form para una mejor experiencia de usuario.

- BIBLIOGRAFÍA: Estándar para validaciones complejas y diseño minimalista de CRUD.

6. FLUJO DE TRABAJO VALIDADO PARA NUEVOS MÓDULOS
------------------------------------------------
Este es el proceso probado para crear módulos con cero fricción:
1. **Analizar Base de Datos**: Identificar tabla y claves.
2. **Capa de Datos (DRP)**: Crear `IndexRepo`, `CreateRepo`, `EditRepo`, `ViewRepo` usando Query Builder.
3. **Capa Lógica (Forms)**: Crear `Create[Modulo]Form` y `Update[Modulo]Form`.
   - *Clave*: Usar el Regex Universal y validación `unique` ignorando ID en update.
4. **Capa de Control (Livewire)**:
   - `List[Modulo]`: Paginación y búsqueda.
   - `Create[Modulo]`: Guardado sin redirección (UX fluida).
   - `Update[Modulo]`: Carga en `mount()` y redirección al guardar.
   - `Show[Modulo]`: Solo lectura.
5. **Capa Visual (Blade)**: Clonar vistas de PNF/Evaluaciones respetando clases CSS de Tailwind.
6. **Integración**:
   - `web.php`: Agrupar rutas.
   - `side-bar.blade.php`: Añadir menú desplegable con icono de Material Symbols.
